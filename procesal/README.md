# Procesal — Derecho procesal

Este módulo asiste en la preparación y gestión de procesos judiciales en todas las jurisdicciones panameñas: civil, penal, contencioso-administrativo y laboral. Cubre la elaboración de cronologías de hechos, borradores de demanda, análisis de requerimientos recibidos, cómputo y control de términos procesales conforme al Código Judicial de Panamá, intake de nuevos asuntos, recursos (apelación, casación ante la Corte Suprema de Justicia), briefings ejecutivos y visión de portfolio. Pensado para litigantes, apoderados judiciales y departamentos de contencioso.

## Skills disponibles

| Skill | Comando | Descripción |
|-------|---------|-------------|
| Cronología | `/cronologia` | Genera una línea temporal de hechos relevantes a partir de documentos del caso |
| Demanda | `/demanda` | Prepara un borrador estructurado de demanda con hechos, fundamentos y petitum |
| Requerimiento recibido | `/requerimiento-recibido` | Analiza un requerimiento o demanda recibida e identifica plazos y opciones de respuesta |
| Plazos | `/plazos` | Calcula y controla términos procesales según jurisdicción y tipo de proceso |
| Intake | `/intake` | Estructura la información inicial de un nuevo asunto litigioso |
| Briefing | `/briefing` | Genera un resumen ejecutivo del estado del procedimiento para cliente o socio |
| Portfolio | `/portfolio` | Vista general del estado de todos los asuntos litigiosos activos |

## Primeros pasos

1. Ejecuta la entrevista inicial del proyecto para configurar tu perfil y preferencias.
2. Prueba `/cronologia` con la documentación de un caso para generar la línea temporal de hechos.
3. Usa `/plazos` para verificar los términos procesales de un proceso en curso.
4. Ejecuta `/intake` cuando recibas un nuevo asunto para estructurar toda la información desde el primer día.

## Casos de uso típicos

- Preparación de la cronología de hechos para un proceso ordinario civil.
- Análisis de una demanda recibida para identificar términos de contestación y posibles excepciones.
- Control centralizado de términos procesales de todos los asuntos del despacho.
- Generación de un briefing para el cliente sobre el estado de su proceso.

## Lo que este plugin no hace

- **No presenta escritos ante los tribunales**: no firma ni envía documentos procesales a través del sistema de notificaciones del Órgano Judicial ni ningún otro sistema.
- **No sustituye al apoderado judicial**: no asume funciones de representación procesal.
- **No calcula costas procesales**: no emite tasaciones de costas ni minutas de honorarios.
- **No sustituye el criterio profesional** del abogado: los borradores deben ser revisados y firmados por un abogado idóneo (con tarjeta profesional vigente).
